# ระบบสร้างหน้า SEO + sitemap อัตโนมัติ สำหรับ CHIRASIN MARKET

ระบบนี้ทำให้ Google มีโอกาสค้นเจอ **แต่ละประกาศแยกกัน** ในผลค้นหา (ไม่ใช่แค่หน้าแรก) โดยทำงานอัตโนมัติผ่าน GitHub Actions ทุกวัน — ไม่ต้องมี server เอง ไม่มีค่าใช้จ่ายเพิ่ม

## มีไฟล์อะไรบ้าง

```
📁 repo ของคุณ (chirasin99.github.io)
├── index.html                              ← ไฟล์เดิมที่มีอยู่แล้ว ไม่ต้องแตะ
├── package.json                            ← ไฟล์ใหม่ (บอกว่าต้องใช้ Firebase SDK)
├── scripts/
│   └── generate-static-pages.js            ← ไฟล์ใหม่ (สคริปต์หลัก)
└── .github/
    └── workflows/
        └── generate-seo-pages.yml          ← ไฟล์ใหม่ (ตั้งเวลารันอัตโนมัติ)
```

พอรันแล้วจะมีไฟล์เกิดขึ้นใหม่ (ไม่ต้องสร้างเอง สคริปต์สร้างให้อัตโนมัติ):
```
├── ad/
│   ├── 1783748234662-fgcsI0/
│   │   └── index.html                      ← หน้า SEO ของประกาศแต่ละอัน
│   └── ...
└── sitemap.xml                             ← แผนที่ลิงก์ทั้งหมดให้ Google
```

## วิธีติดตั้ง (ทำครั้งเดียว)

1. **อัปโหลดไฟล์ทั้งหมดในนี้** ไปไว้ใน repo `chirasin99.github.io` ตามโครงสร้างโฟลเดอร์ด้านบน
   (ลาก-วางผ่านหน้าเว็บ GitHub ก็ได้ หรือใช้ GitHub Desktop/Git ก็ได้)

2. **เปิดสิทธิ์ให้ GitHub Actions เขียนกลับเข้า repo ได้**
   - ไปที่ repo → **Settings** → **Actions** → **General**
   - เลื่อนลงไปหา **Workflow permissions**
   - เลือก **"Read and write permissions"**
   - กด **Save**

   (ถ้าข้ามขั้นตอนนี้ ระบบจะสร้างไฟล์ได้ แต่ push กลับเข้า repo ไม่ได้ จะ error ตรง step สุดท้าย)

3. **รันครั้งแรกด้วยตัวเอง เพื่อทดสอบว่าทำงานถูกต้อง**
   - ไปที่แท็บ **Actions** บนหน้า repo
   - เลือกเวิร์กโฟลว์ชื่อ **"สร้างหน้า SEO และ sitemap อัตโนมัติ"** ทางซ้าย
   - กดปุ่ม **"Run workflow"** (มุมขวา) → กด **Run workflow** อีกที
   - รอสัก 1-2 นาที แล้วดูว่าสถานะขึ้นติ๊กถูกสีเขียวไหม

4. **เช็คผลลัพธ์**
   - เข้า repo ดูว่ามีโฟลเดอร์ `ad/` กับไฟล์ `sitemap.xml` เกิดขึ้นใหม่ พร้อม commit ข้อความ "🤖 อัปเดตหน้า SEO และ sitemap อัตโนมัติ"
   - ลองเปิด `https://chirasin99.github.io/sitemap.xml` ดูว่าโชว์ลิงก์ประกาศครบไหม
   - ลองเปิดลิงก์ประกาศอันใดอันหนึ่งจาก sitemap ดู ควรจะเห็นเนื้อหาสินค้าแวบหนึ่งแล้วเด้งไปหน้าเว็บแอปปกติทันที

5. **ส่ง sitemap ใหม่ให้ Google**
   - เข้า Google Search Console → เมนู **Sitemap**
   - ใส่ `sitemap.xml` แล้วกด **ส่ง**
   - (sitemap เดิมที่เคยส่งไว้แบบ manual สามารถลบทิ้งได้ เพราะตอนนี้อันใหม่จะอัปเดตอัตโนมัติทุกวันแทน)

## หลังจากนี้ไม่ต้องทำอะไรเพิ่ม

ระบบจะรันอัตโนมัติทุกวัน (ตั้งไว้ที่ 20:00 น. เวลาไทย ในไฟล์ `.github/workflows/generate-seo-pages.yml` — แก้เวลาได้ที่บรรทัด `cron:`) โดยจะ:
- ดึงประกาศที่ยังไม่หมดอายุ/ไม่ถูกปิดทั้งหมดจาก Firestore
- สร้างหน้า SEO ใหม่ให้ประกาศที่เพิ่งลงเพิ่ม
- ลบหน้าของประกาศที่หมดอายุ/ถูกปิดไปแล้วออกอัตโนมัติ
- อัปเดต sitemap.xml ให้ตรงกับสถานะล่าสุดเสมอ

ถ้าอยากให้รันทันทีหลังลงประกาศใหม่ ไม่ต้องรอถึงรอบถัดไป ก็กด **Run workflow** เองได้ตลอดเวลาตามขั้นตอนข้อ 3

## ถ้าเจอปัญหา

เปิดแท็บ **Actions** → คลิกที่ run ที่ error (มีกากบาทสีแดง) → ดู log ว่า error ตรงไหน ส่งข้อความ error มาให้ดูได้เลยครับ
